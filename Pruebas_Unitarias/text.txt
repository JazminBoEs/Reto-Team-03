README - Pruebas Unitarias
==========================

Este documento explica cómo ejecutar las pruebas unitarias del backend de forma independiente.

Requisitos previos:
-------------------
- Python 3.8+ instalado.
- Estar ubicado en la carpeta raíz del proyecto.
- Contar con las dependencias de backend necesarias.

Dependencias necesarias:
------------------------
Las pruebas unitarias utilizan Flask y pytest, y el backend importa módulos de Flask y MySQL.
Instala las dependencias con el siguiente comando:

```bash
cd /workspaces/Reto-Team-03
python -m pip install --upgrade pip
python -m pip install pytest flask flask-cors mysql-connector-python werkzeug
```

Ejecutar las pruebas:
---------------------
1. Ve a la carpeta raíz del proyecto:
   ```bash
   cd /workspaces/Reto-Team-03
   ```
2. Ejecuta el archivo de pruebas con pytest:
   ```bash
   pytest Pruebas_Unitarias/test_app.py -v
   ```
3. Si todo está correcto, deberías ver una salida similar a `== X passed in ...` y un `exit code 0`.

Qué se prueba:
--------------
- Endpoints del backend mediante el cliente de pruebas `app.test_client()`.
- Rutas de `usuarios`, `sensores` y `usuarios-predios`.
- Estados HTTP esperados: `200`, `201`, `204`, `400`, `404`, `500`.
- Validaciones de login y creación de usuarios.
- Parcheo de `main.execute_query` con `unittest.mock.patch` para evitar llamadas reales a la base de datos.

Notas importantes:
------------------
- No es necesario levantar el servidor Flask en modo normal para ejecutar estas pruebas.
- Las pruebas usan `app.test_client()`, por lo que las solicitudes se simulan internamente.
- Si aparece un error de importación, instala las dependencias del backend con `python -m pip install ...`.
- Ejecuta pytest desde la raíz del proyecto para que `Pruebas_Unitarias/conftest.py` pueda importar `main.py` correctamente.
